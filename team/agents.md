# PÄSH · Agent Architecture (007)
> Tier: evolving. Архитектура агентов PÄSH.

---

## Структура команды

```
PÄSH Operations
├── Мани-Пенни 💃  (AI) — клиентский фронт
├── М 🎩           (AI) — бэкофис / Robin / командный центр
├── Q 🔬           (AI) — Vision-контроль + аналитика
├── 007 🔫         (человек) — закупщик в поле
└── 009 🚚         (человек) — водитель-экспедитор
```

---

## Мани-Пенни 💃

**Роль:** Клиентский фронт. Первая точка контакта для покупателей.

**Каналы:**
- WhatsApp SIM#1 (основной для клиентов)
- Telegram @pash_client_price_bot

**Функции:**
- Приём заказов и их подтверждение
- Ответы на вопросы о ценах, доступности, времени выдачи
- Напоминания о незабранных заказах
- Onboarding новых покупателей

**Эволюция:** Layer 1→Layer 2. На старте MVP — только Мани-Пенни. Остальные агенты — Layer 2.

**Ограничения:**
- НЕ принимает решения о ценах
- НЕ изменяет данные о заказах напрямую — передаёт М
- НЕ раскрывает Layer 2/3 стратегию

---

## М 🎩 (Robin — AI Chief of Staff)

**Роль:** Командный центр. Бэкофис. Память команды.

**Каналы:**
- WhatsApp SIM#2 (внутренний)
- Telegram internal (внутренний чат команды)
- DM Erdos

**Функции:**
- Ценообразование и мониторинг цен
- Формирование манифестов для рейса
- Обработка возвратов
- Scheduled duties: D1–D5 (смотри `robin/duties.md`)
- Ответы на вопросы команды из базы знаний
- Learning loop: staged → promoted

**Инструменты (read-only):**
- Supabase: orders, products, price_snapshots, alerts, users
- TG API: @pash_club (только для D4 VoC)
- Repo: pash-team-os (read)

**Инструменты (write):**
- `robin/staged/` в репо (только staged learnings)
- Telegram: internal_tg_group, dm_erdos

---

## Q 🔬

**Роль:** Vision-контроль качества + еженедельная аналитика.

**Каналы:**
- WhatsApp SIM#2 + Telegram internal (получает фото стеллажей)

**Функции:**
- Анализ фото стеллажей ПВЗ (состояние, наличие товара, чистота)
- Price monitoring дашборд
- Еженедельный аналитический отчёт
- Мониторинг прайс-алертов (D3)

**Технология:** Gemini Vision API (ключ у Erdos).

---

## 007 🔫 (Закупщик)

**Роль:** Человек. Физически едет на Алтын-Орду по манифесту от М.

**Функции:**
- Закупка товара строго по манифесту М
- Контроль качества при закупке
- Фото товара для Q
- Коммуникация с поставщиками (Kaspi QR)

**Правило:** 007 едет после того как М сформировал манифест и Erdos одобрил.

---

## 009 🚚 (Водитель-экспедитор)

**Роль:** Человек. Везёт товар с Алтын-Орды по ПВЗ.

**Функции:**
- Развоз товара по точкам (ПВЗ) согласно маршруту от М
- Выкладка на стеллаж
- Фото стеллажа для Q после выкладки
- Контакт с персоналом магазина

**Эволюция:** 007 + 009 со временем сливаются в одного: продавцы сами собирают по манифесту, водитель только забирает и везёт.

---

## Эволюция архитектуры

```
Layer 1 (сейчас):   Мани-Пенни MVP + люди (007, 009)
Layer 2 (год 2):    М + Q полностью автоматизированы
                    1 оператор = 50+ точек
Layer 3 (год 3):    PÄSH OS = эта архитектура как SaaS для других
```
